# Yango Competitive Intelligence — MVP

## Функциональные блоки

### 1. Сбор данных

#### 1.1 Парсинг сайтов конкурентов
- Официальные сайты: InDriver, Didi, Uber, Cabify (Перу)
- Страницы тарифов для водителей
- Страницы тарифов для пассажиров
- Страницы промо-акций

#### 1.2 Парсинг магазинов приложений
- App Store: релиз-ноты, рейтинг, количество отзывов
- Google Play: релиз-ноты, рейтинг, количество отзывов
- История обновлений за последние 6 месяцев

#### 1.3 Сбор отзывов
- Отзывы из App Store и Google Play
- Сегментация: водители / пассажиры
- Минимум 100 отзывов на конкурента

---

### 2. Структура данных

#### 2.1 Сущности
- **Конкурент**: название, регион, ссылки на источники
- **Тариф**: тип (водитель/пассажир), значение, дата фиксации, источник
- **Промо**: описание, условия, даты, источник
- **Релиз**: версия, дата, описание изменений, платформа
- **Отзыв**: текст, рейтинг, дата, роль (водитель/пассажир), источник

#### 2.2 Классификация продуктовых изменений
Категории:
- Тарифы и комиссии
- UX/UI
- Безопасность
- Водительский опыт
- Пассажирский опыт
- Промо и маркетинг

---

### 3. Выходные данные

#### 3.1 База данных
- Все собранные данные в структурированном виде
- Связи между сущностями
- Метаданные: дата сбора, источник, достоверность

#### 3.2 Сравнительные таблицы
- Тарифы водителей: Yango vs конкуренты
- Тарифы пассажиров: Yango vs конкуренты
- Фичи: матрица наличия по конкурентам
- Промо: текущие акции по конкурентам

#### 3.3 Аналитический отчёт
- Сильные/слабые стороны каждого конкурента
- Гипотезы для продуктовой команды (3-5 штук)

---

### 4. Документация

- Список источников с URL
- Описание структуры данных
- Инструкция по повторному сбору данных
