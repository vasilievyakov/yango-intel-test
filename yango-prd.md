# Yango Competitive Intelligence — PRD

## Функциональные требования

### FR-001. Парсинг сайтов конкурентов
- Официальные сайты: InDriver, Didi, Uber, Cabify
- Извлечение: тарифы водителей, тарифы пассажиров, условия работы
- Частота: по запросу (MVP), еженедельно (автоматизация)

### FR-002. Парсинг App Store / Google Play
- Релиз-ноты приложений
- Рейтинг и количество отзывов
- История обновлений

### FR-003. Сбор отзывов пользователей
- Источники: App Store, Google Play
- Сегментация: водители / пассажиры
- Атрибуты: текст, рейтинг, дата

### FR-004. Мониторинг публичных чатов
- Telegram-чаты водителей (публичные)
- Приоритет: Should Have

### FR-005. Мониторинг новостей
- Пресс-релизы и новости о конкурентах
- Приоритет: Should Have

### FR-006. Парсинг закрытых чатов
- Telegram-чаты водителей (закрытые)
- Приоритет: Nice to Have (вне MVP)

---

## Требования к анализу

### FR-007. Сравнение тарифов водителей
- Комиссии платформ
- Бонусы и условия

### FR-008. Сравнение тарифов пассажиров
- Базовые тарифы
- Surge pricing (если доступно)

### FR-009. Отслеживание промо-акций
- Текущие скидки и акции
- Условия и сроки

### FR-010. Классификация изменений
Категории:
- Тарифы и комиссии
- UX/UI
- Безопасность
- Водительский опыт
- Пассажирский опыт
- Промо и маркетинг

### FR-011. Приоритизация изменений
- Оценка влияния на бизнес
- Приоритет: Should Have

### FR-012. Сегментация отзывов
- По ролям: водители / пассажиры
- По темам: цена, качество, поддержка
- Приоритет: Should Have

---

## Требования к выходным данным

### FR-013. Еженедельный дайджест
- Список изменений за неделю
- Группировка по конкурентам и категориям

### FR-014. Сравнительные таблицы
- Тарифы: Yango vs конкуренты
- Фичи: матрица наличия

### FR-015. Гипотезы для продукта
- 3-5 actionable гипотез в неделю
- Связь с данными

### FR-016. Алерты
- Уведомления о критичных изменениях
- Приоритет: Should Have

### FR-017. Дашборды
- Интерактивная визуализация
- Приоритет: Nice to Have (вне MVP)

---

## Требования к инфраструктуре

### FR-018. База данных
Сущности:
- Конкурент (название, регион, источники)
- Тариф (тип, значение, дата, источник)
- Промо (описание, условия, даты)
- Релиз (версия, дата, изменения, платформа)
- Отзыв (текст, рейтинг, дата, роль)
- Изменение (категория, описание, дата, конкурент)

### FR-019. Документация
- Источники данных
- Структура БД
- Инструкции по сбору

### FR-020. Архитектура для автоматизации
- Модульная структура парсеров
- API для доступа к данным
- Приоритет для MVP: проектирование, не реализация
